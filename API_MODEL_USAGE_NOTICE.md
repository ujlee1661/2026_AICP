# API Model Usage Notice

## 핵심 원칙

API 호출에 사용하는 모델은 반드시 `.env` 파일에 작성된 모델만 사용한다.

## 주의사항

- `.env`에 지정된 API 모델 외에 임의로 다른 모델을 선택하거나 대체하지 않는다.
- 실행 속도가 느리더라도 더 빠른 모델, 더 저렴한 모델, 기본 모델 등으로 자동 변경하지 않는다.
- 코드, 스크립트, 설정 파일에서 모델명을 직접 하드코딩하지 않는다.
- 모델 변경이 필요하면 먼저 `.env`의 모델 설정을 수정한 뒤 실행한다.
- fallback 로직을 추가하더라도 다른 모델로 전환하는 방식은 사용하지 않는다.
- API 실패, 속도 저하, rate limit 문제가 발생해도 허가 없이 모델을 바꾸지 않는다.

## 운영 기준

시뮬레이션, 뉴스 해석, 에이전트 판단, 리포트 생성 등 모든 API 기반 작업은 `.env`에 명시된 모델 설정을 단일 기준으로 사용한다.

성능이나 속도보다 실험 조건의 일관성이 우선이다. 따라서 실행 시간이 길어지더라도 요청된 API 모델만 사용해야 한다.
